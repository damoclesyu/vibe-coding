# RSS 智能阅读器（国际局势版）— 设计文档

## 1. 项目背景与目标

### 1.1 背景
用户通过浏览器获取国际局势信息时，常遇到以下问题：
- 信息分散、广告及推荐内容干扰严重
- 难以快速判断信息可信度（谣言、立场偏颇、断章取义）
- 多源信息缺乏交叉验证

### 1.2 目标
构建一个 RSS 智能阅读器，专注于每日国际局势新闻：
- 多源聚合（预置 10 个左右权威信源）
- AI 自动生成一句话中文摘要（≤40 字）
- AI 输出可信度评分（0-100）及理由
- 干净、可筛选的阅读界面

### 1.3 Demo 范围限定
- 预置固定 RSS 源，不支持用户动态增删
- 数据抓取与 AI 处理为手动运行脚本
- 前端纯静态页面，通过 `fetch` 读取本地生成的 JSON 文件
- 所有排序/筛选由前端 JavaScript 完成，无后端 API

---

## 2. 用户故事与核心流程

### 2.1 用户故事
> 作为一位关注国际局势的普通读者，我希望打开一个网页就能看到今日最重要的国际新闻，每条新闻被 AI 概括核心内容、打上可信度分数，我能优先阅读高可信内容，避免被低质文章浪费时间。

### 2.2 核心流程
1. 运行数据脚本 → 从多个 RSS 源抓取最新文章
2. 对每篇文章调用 AI API → 生成摘要 + 评分 + 理由
3. 结果保存为 `articles.json`
4. 用户打开前端页面 → 自动加载 JSON → 展示卡片列表
5. 用户可按「可信度优先」或「最新优先」排序，可筛选只显示高可信文章

```json
[

    "link": "https://reuters.com/example",
    "pubDate": "2026-05-19T08:30:00Z",
    "contentSnippet": "RSS 原始摘要内容...",

    "credibilityLevel": "high",
    "credibilityReason": "来源权威、有直接引述、多方交叉验证、时间地点明确"
```

## 5. 技术架构

### 5.1 数据流

```
[RSS源] → [抓取脚本] → [去重 & 清洗] → [调用AI API] → [articles.json] → [前端加载展示]
```

### 5.2 技术栈

| 层 | 技术 | 理由 |
|----|------|------|
| 脚本环境 | Node.js | 统一语言，`rss-parser` 成熟稳定 |
| RSS 解析 | `rss-parser` | 支持 RSS/Atom |
| AI 接口 | Claude API（兼容 OpenAI 格式） | 已配置好，成本可控 |
| 数据存储 | JSON 文件（`articles.json`） | 零配置，适合 Demo |
| 前端 | 原生 HTML/CSS/JS | 无需构建，直接打开或轻量服务器启动 |
| 部署 | 本地 `npx serve` 或静态托管 | 简单高效 |

### 5.3 组件树

```
页面
├── Header
│   ├── 标题（📰 RSS 智能阅读器）
│   ├── 更新状态标签（数据更新时间）
│   └── FilterBar（可信度 ↓ / 最新 / 仅高可信开关）
├── NewsList
│   ├── DateSeparator（📅 今天 — 5月19日 · 8 条）
│   └── NewsCard × N
│       ├── 来源 + 时间行
│       ├── 标题（可点击跳转原文）
│       ├── CredibilityBadge（可信度胶囊标签）
│       ├── AI 摘要（浅灰背景 + 左侧蓝色竖线）
│       └── 评分理由（小字灰色）
└── Footer（"AI 摘要 & 可信度评分 | 每日更新国际局势"）
```

## 6. 去重算法

按优先级执行两层去重：

**第一层 — URL 精确匹配：**
- 同一 URL 的文章视为重复，只保留首次抓取的记录

**第二层 — 标题相似度匹配（兜底）：**
- 不同 URL 但标题 Jaccard 相似度 > 0.8 视为转载同一事件
- 匹配后保留发布时间最早的记录

## 7. AI 提示词模板

**系统提示词：**

```
你是一位国际新闻分析师，擅长快速评估新闻可信度。

请分析以下新闻文章，输出严格的 JSON 格式（不要输出其他内容）：

{
  "summary": "一句话中文摘要，不超过40字，客观冷静的语气",
  "credibilityScore": 85,
  "credibilityLevel": "high",
  "credibilityReason": "简要说明评分依据（50字以内）"
}

评分标准（0-100 分）：
- 90-100：来源权威（路透/BBC/新华社等一线通讯社）、有直接引述、多源交叉验证
- 70-89：来源较权威、内容具体、事实可查证
- 50-69：知名媒体但非一线、报道简短、部分细节不完整
- 30-49：来源不明、缺少署名、情绪化用语较多
- 0-29：明显谣言、完全无来源、煽动性标题

评分时考虑：
1. 来源权威性（权重最高）
2. 信息具体程度（时间/地点/人物/数据）
3. 是否有直接引述或原始出处
4. 用词是否客观中性
5. 能否与其他来源交叉验证
```

**API 参数：**
- model: claude-sonnet-4-6
- max_tokens: 300
- temperature: 0.3
- response_format: JSON mode 强制结构化输出

## 8. Demo 验收标准

1. **抓取脚本**：能成功运行，生成 `articles.json`，包含至少 15 条有效新闻
2. **AI 处理**：每条新闻有合理的摘要、分数（0-100）、理由（不空泛）
3. **前端展示**：打开页面正常显示新闻列表，卡片信息完整
4. **排序功能**：可信度优先按分数从高到低排列；最新优先按时间从新到旧排列
5. **筛选功能**：点击「仅高可信」只显示分数 ≥80 的文章
6. **性能**：页面加载无明显卡顿（50 条以内渲染 < 1 秒）

## 9. 风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 部分 RSS 源无法访问 | 抓取文章数量减少 | 预置备用源，脚本跳过失败源并打印日志 |
| AI API 超时或限流 | 部分文章无 AI 结果 | 设置 10 秒超时，失败后填入默认值继续 |
| API 费用超预算 | 开发成本 | Demo 阶段限制总处理 ≤30 篇 |
| 跨域问题 | `file://` 协议可能被 CORS 阻止 | 使用 `npx serve` 或 VS Code Live Server |
