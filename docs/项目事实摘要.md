# 国际局势智能简报 — 项目事实摘要

> 用途：将本文档提供给 AI，生成简历文案、面试话术、作品集说明等。

---

## 一句话描述

一款 AI 驱动的国际新闻聚合工具，自动抓取多源 RSS → DeepSeek 大模型生成摘要和可信度评分 → 前端卡片流展示。每天 8:00 自动更新，浏览器打开即用，零注册零配置。

在线地址：https://damoclesyu.github.io/vibe-coding/

---

## 技术实现

| 层级 | 用了什么 | 怎么用的 |
|------|---------|---------|
| 数据获取 | rss-parser (Node.js) | 抓取 8 个国内 RSS 源，每源取前 10 条，按 URL 去重 |
| AI 处理 | DeepSeek Chat API | 对每条新闻生成 40 字中文摘要 + 0-100 可信度评分 + 50 字评分理由 |
| 前端 | 原生 HTML/CSS/JS | 单文件应用，无框架无依赖，卡片流布局 + 响应式设计 |
| 存储 | JSON 静态文件 | 预生成 articles.json，前端 fetch 读取，无后端无数据库 |
| 用户数据 | localStorage | 自定义 RSS 源、隐藏的信源列表保存在浏览器本地 |
| 自动任务 | GitHub Actions | 每天 UTC 0:00（北京时间 8:00）自动执行抓取脚本 |
| 部署 | GitHub Pages | 推送即部署，全球 CDN，自定义域名不需要 |

## 数据管道全流程

```
每天 8:00 GitHub Actions 触发
  → npm install (安装 rss-parser)
  → node scripts/fetch.js 执行:
      → 遍历 8 个 RSS 源，抓取文章
      → 过滤：只保留发布时间在最近 24 小时内的文章
      → URL 去重
      → 取前 30 条，逐一调用 DeepSeek API
      → AI 返回 JSON：{ summary, credibilityScore, credibilityLevel, credibilityReason }
      → AI 失败时回退：score = 来源权重 × 0.6
      → 输出 articles.json（含 generatedAt 时间戳 + 总文章数统计）
  → git commit + push（如有数据变更）
  → 自动部署到 GitHub Pages
```

## 前端功能清单

- 新闻卡片流：标题、来源、时间、AI 摘要、可信度 Badge、评分理由（低可信度才显示）
- 两种排序：按最新、按可信度；每种支持升降序
- 搜索：标题、摘要、正文、来源名称实时过滤
- 仅高可信开关：一键过滤 <80 分的文章
- 日期分组：今天 / 昨天 / 更早
- 双时间戳：页面打开时间（动态）+ 数据更新时间（来自 articles.json）
- 侧边栏：高/中/低可信度统计 + 比例条 + 信源列表
- 信源管理：内置源可隐藏/恢复，支持添加自定义 RSS 源
- 移动端适配：≤860px 切换为单栏布局

## AI 行为设计细节

评分 Prompt 包含：
- 5 个评分维度：来源权威性、信息具体程度、原始出处可追溯性、用词客观性、交叉验证可行性
- 5 档评分区间：90-100（顶级通讯社、多源交叉验证）、70-89（较权威、事实可查）、50-69（知名媒体但非一线）、30-49（来源不明、情绪化）、0-29（明显谣言）
- 输出格式：严格 JSON，字段为 summary(≤40 字)、credibilityScore(数字)、credibilityLevel(high/medium/low)、credibilityReason(≤50 字)

容错三层降级：
1. API 正常返回 → 解析 JSON，写入各字段
2. API 返回但无法解析 JSON → 正则提取 JSON 块，仍失败则抛错
3. API 调用失败 → 分数 = 来源权重 × 0.6，标记"AI 处理失败"
