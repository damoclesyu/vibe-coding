## 一、项目简介

这是一款 **AI 驱动的国际新闻聚合工具**。每天自动抓取多个权威媒体 RSS 源，用大模型对每条新闻生成一句话摘要和可信度评分（0-100），最终呈现在一个干净的信息卡片流页面上。

核心价值：**3 分钟掌握当日要闻，一眼识别信息可信度**。

解决的问题：

| 痛点 | 解决方案 |
|------|----------|
| 多源信息分散，逐个打开效率低 | 自动聚合 8 个权威源，一站式浏览 |
| 标题党泛滥，难以判断真假 | AI 五维度评分，用颜色编码区分可信度等级 |
| 新闻量大，找不到重点 | AI 一句话摘要，快速筛选感兴趣的内容 |

## 二、产品功能清单

### 数据层

- **8 个预置 RSS 源**（人民网、新华网、央视网、中国日报、凤凰网、中国新闻网等），全部国内直连
- **每日 8:00 自动抓取**（GitHub Actions 定时任务），无需人工干预
- **24 小时时间窗口**：只保留前一天 8:00 至当天 8:00 的新闻，过滤陈旧内容
- **URL 去重**：跨源重复新闻自动合并
- **降级策略**：AI 处理失败时，回退到来源权重 × 60% 作为默认可信度

### AI 处理层

- **大模型**：DeepSeek（成本为 Claude 的 1/10，中文理解能力优秀）
- **评分体系**：5 维度评估（来源权威性、信息具体程度、原始出处可追溯性、用词客观性、交叉验证可行性）
- **5 档评分区间**：90-100 / 70-89 / 50-69 / 30-49 / 0-29
- **结构化输出**：严格 JSON 格式，40 字摘要 + 50 字评分理由

### 前端展示层

- **卡片流布局**：每条新闻独立成卡，整个卡片可点击跳转原文
- **可信度 Badge**：绿（高 ≥80）/ 橙（中 60-79）/ 红（低 <60）三色编码
- **评分理由仅低可信显示**：减少视觉干扰，只在需要警惕时才展示理由
- **日期分组**：今天 / 昨天 / 更早，方便按时间浏览
- **双时间戳**：页面打开时间 + 数据更新时间，用户知道新闻是何时抓取的
- **侧边栏统计**：高/中/低可信度数量 + 比例条 + 信源列表

### 交互层

- **排序**：按最新 / 按可信度，支持升降序切换
- **搜索**：标题、摘要、正文、来源名称实时过滤
- **仅高可信开关**：一键过滤低质量内容
- **信源管理**：内置源可隐藏/恢复，支持添加自定义 RSS 源
- **响应式设计**：桌面端双栏布局，移动端单栏

### 工程层

- **GitHub Pages 部署**：推送即上线，全球 CDN
- **自动部署链**：数据抓取 → AI 处理 → 数据提交 → Pages 部署，全自动
- **零后端**：纯静态页面 + 预生成 JSON，无服务器成本

## 四、产品设计思考（面试可展开）

### 4.1 AI 行为设计：Prompt 就是给 AI 的 PRD

这个项目的 Prompt 本质上是一份 **AI 产品需求文档**：

- **评分标准** = 功能需求（AI 应该怎么判断）
- **输出格式约束** = 接口定义（AI 应该输出什么）
- **容错降级** = 异常处理（AI 出错时怎么办）

做 AI PM 的核心能力之一，就是把模糊的"智能"需求翻译成 AI 能稳定执行的规则。

### 4.2 AI 产品的容错设计

AI 是概率系统，永远需要 fallback。这个项目的容错链路：

```
API 调用超时/格式异常 → 正则提取兜底 → 仍失败则回退来源权重 × 0.6
```

每一层都有体验兜底，用户不会看到报错或空白卡片。

### 4.3 信息架构决策

| 决策 | 理由 |
|------|------|
| 评分理由仅低可信度显示 | 减少视觉噪音，高可信不需要解释 |
| 默认按最新排序（非可信度） | 新闻的时效性比可信度排序更重要 |
| 卡片整体可点击 | 降低交互成本（费茨定律） |
| 侧边栏放统计 + 信源 | 信息透明，让用户知道数据从哪来 |
| 搜索框在最右侧 | 视线动线：先看排序 → 再到最右侧搜索 |

### 4.4 迭代历程（18 次提交）

| 阶段 | 关键迭代 |
|------|----------|
| 核心链路 | RSS 抓取 → AI 摘要评分 → 前端展示 |
| 体验优化 | 日期分组、卡片点击、评分理由条件展示、最新/可信度排序切换 |
| 可靠性 | AI 降级策略、URL 去重、24 小时时间窗口 |
| 信息透明 | 双时间戳、统计面板、信源列表 |
| 用户控制 | 搜索、仅高可信开关、信源隐藏/恢复、自定义 RSS |
| 分发 | GitHub Pages 部署、每日 8:00 自动抓取 + 自动部署 |
