# AGENTS.md — AI 开发协作规范

本文档供后续 AI Agent 读取，确保所有 AI 协作时行为一致、目录整洁。

---

## 目录结构与用途

| 目录 | 用途 | 规则 |
|------|------|------|
| `src/` | **代码区**，存放所有源代码 | 禁止引用 `docs/`、`notes/` 等外部目录的文件；所有 import 路径限定在本目录内 |
| `docs/` | **产品文档区**，存放 PRD、需求迭代记录、关键架构决策 | 仅 Markdown 或 PDF；按 `YYYY-MM-DD_标题.md` 命名以保持时间线可追溯 |
| `assets/design/` | **设计素材**，效果图、UI 参考稿 | 图片格式为主，不要放代码 |
| `assets/bug/` | **Bug 截图**，测试中发现的报错、异常截图 | 按 `YYYY-MM-DD_问题简述.png` 命名 |
| `assets/reference/` | **参考图**，灵感收集、竞品截图 | 非项目产出，仅供设计/开发参考 |
| `notes/` | **学习笔记**，踩坑记录、技术方案调研 | 每篇一个主题；文件名格式 `主题名.md` |
| `.claude/` | Claude Code 本地配置 | 由 `.gitignore` 排除，不上传仓库 |

---

## 开发规则

### 1. 代码隔离
- `src/` 内代码不引用 `docs/`、`notes/`、`assets/` 中的文件
- 代码内所有相对路径限定在 `src/` 目录树内

### 2. 文档归位
- 所有 .md 文档放在 `docs/` 或 `notes/`
- **不要在根目录创建新的 .md 文件**（除 README.md 和 AGENTS.md 外）

### 3. 命名规范
- 文件名使用英文 + 下划线，不用中文命名
- 文档以日期前缀命名：`docs/2026-05-19_PRD_v1.md`
- 代码文件夹按功能模块命名：`src/auth/`、`src/api/`、`src/utils/`

### 4. 提交规范
- 提交前确保 `.gitignore` 已排除敏感文件
- 不提交 `.env`、`node_modules/`、编译产物
- Commit message 用中文简要描述改动原因

### 5. 新建功能的流程
1. 需求文档放在 `docs/`
2. 设计稿放在 `assets/design/`
3. 所有代码在 `src/` 内完成
4. 过程中学到的新知识记录在 `notes/`

---

## 技术约束

- 当前项目初始阶段，技术栈未定，所有技术选型决策记录在 `docs/decisions/`
- 不要引入不必要的第三方依赖
- 优先使用项目已有的工具链和库
